### The Ten Cycles (v24 → v33)

**v24 – Byte-Level Language Interface**  
Add a byte-pair encoding (BPE) tokenizer and a text generation head. The model now outputs bytes, not just numbers. We keep the program execution engine as a separate "tool" that can be called.

**v25 – Unified Reasoning and Language Objective**  
Train on two alternating tasks: (1) text continuation (language modeling) on a small curated corpus of reasoning-heavy text, and (2) program induction from examples. The model learns to switch modes based on input format.

**v26 – Memory-Augmented Context**  
Expand the differentiable memory tape to also store token embeddings for long contexts. The model can read/write both numbers and text embeddings.

**v27 – Self‑Play Curriculum for Language**  
Generate language tasks procedurally: e.g., “What is 2+3?” with answer “5”. The model learns to map linguistic queries to program execution.

**v28 – Sparse Mixture of Experts (MoE) for Capacity**  
To increase parameter count without blowing up FLOPs, add a sparse MoE layer with 4 experts, only 1 active per token. This lets us grow to ~10M parameters.

**v29 – Adaptive Computation Time for Deeper Reasoning**  
Enhance the loop primitive to allow variable‑depth reasoning per example. The model can spend more FLOPs on hard problems.

**v30 – Algorithmic Prompt Decomposition**  
Train the model to break complex instructions into sub‑tasks, solve them via program induction, and compose results. This mimics chain‑of‑thought.

**v31 – Tiny World Model for Common Sense**  
Add a small set of learnable “concept embeddings” for everyday objects and actions, trained from a tiny 10MB text dataset. Not full knowledge, but a seed.

**v32 – Quantization and CPU Optimisation**  
Quantize weights to 8‑bit during training (simulated) to save memory and speed up inference, making the model actually runnable on the i5 with decent throughput.

**v33 – Integrated Final System**  
Combine all pieces into a single script: a 10M‑parameter sparse transformer that can (a) carry on a basic conversation, (b) answer simple math/reasoning questions by inducing algorithms in‑context, (c) learn new tasks from a few examples, and (d) run entirely on an i5‑4300 with 8GB RAM.

# ---------- SPARSE MOE LAYER ----------
class MoELayer(nn.Module):
    def __init__(self, dim, num_experts, top_k):
        super().__init__()
        self.top_k = top_k
        self.router = nn.Linear(dim, num_experts)
        self.experts = nn.ModuleList([nn.Sequential(nn.Linear(dim, dim*2), nn.GELU(), nn.Linear(dim*2, dim)) for _ in range(num_experts)])

    def forward(self, x):
        B, T, D = x.shape
        gating = self.router(x)  # (B, T, E)
        weights, indices = torch.topk(gating, self.top_k, dim=-1)
        weights = F.softmax(weights, dim=-1)
        out = torch.zeros_like(x)
        for i, expert in enumerate(self.experts):
            mask = (indices == i).any(dim=-1).unsqueeze(-1).float()
            if mask.sum() > 0:
                expert_out = expert(x)
                out += mask * expert_out * weights[:,:,[i]].expand_as(expert_out)
        return out

# ---------- ADAPTIVE TRANSFORMER BLOCK ----------
class AdaptiveBlock(nn.Module):
    def __init__(self, dim, heads):
        super().__init__()
        self.ln1 = nn.LayerNorm(dim)
        self.attn = nn.MultiheadAttention(dim, heads, batch_first=True)
        self.ln2 = nn.LayerNorm(dim)
        self.moe = MoELayer(dim, NUM_EXPERTS, ACTIVE_EXPERTS)
        self.halt_net = nn.Linear(dim, 1)

    def forward(self, x, memory=None):
        # Self-attention with optional memory as extra keys/values
        if memory is not None:
            # memory: (B, MEM_CELLS, DIM)
            mem_vals = memory.unsqueeze(1).expand(-1, x.shape[1], -1, -1).reshape(-1, memory.shape[1], memory.shape[2])
            # add memory to key/value
            x2, _ = self.attn(self.ln1(x), torch.cat([x, mem_vals], dim=1), torch.cat([x, mem_vals], dim=1))
        else:
            x2, _ = self.attn(self.ln1(x), self.ln1(x), self.ln1(x))
        x = x + x2
        x = x + self.moe(self.ln2(x))
        # Halting score (for ACT)
        halt = torch.sigmoid(self.halt_net(x))
        return x, halt

# ---------- UNIFIED MODEL ----------
class PhaetonV33(nn.Module):
    def __init__(self):
        super().__init__()
        self.token_embed = nn.Embedding(VOCAB_SIZE, EMBED_DIM)
        self.pos_embed = nn.Parameter(torch.randn(1, MAX_SEQ_LEN, EMBED_DIM))
        self.blocks = nn.ModuleList([AdaptiveBlock(EMBED_DIM, NUM_HEADS) for _ in range(NUM_LAYERS)])
        self.ln_f = nn.LayerNorm(EMBED_DIM)
        self.lm_head = nn.Linear(EMBED_DIM, VOCAB_SIZE)

        # Program execution heads (for reasoning tasks)
        self.prog_head = nn.Linear(EMBED_DIM, MAX_PROG_LEN * NUM_PRIMS)
        self.mem_init = nn.Linear(EMBED_DIM, MEM_CELLS * MEM_DIM)

    def forward(self, input_ids, mode='lm', demos=None):
        """
        mode='lm': standard language modeling
        mode='reason': produce program logits for execution
        """
        B, T = input_ids.shape
        x = self.token_embed(input_ids) + self.pos_embed[:, :T, :]
        memory = self.mem_init(x[:,0,:]).view(B, MEM_CELLS, MEM_DIM)  # initial memory from first token
        halts = []
        for block in self.blocks:
            x, halt = block(x, memory)
            halts.append(halt)
        x = self.ln_f(x)

        if mode == 'lm':
            return self.lm_head(x)
        elif mode == 'reason':
            # use last token's representation
            logits = self.prog_head(x[:, -1, :]).view(B, MAX_PROG_LEN, NUM_PRIMS)
            return logits, memory
        return x

def generate_mixed_batch(batch_size):
    # Half batch: language continuation (simple sentences)
    # Half batch: reasoning tasks (arithmetic, list ops)
    lang_inputs = []
    lang_targets = []
    reason_inputs = []
    reason_demos = []
    reason_queries = []
